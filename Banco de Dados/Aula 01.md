
## Aula 1: Atividade em Grupo

### Pesquisa Inicial e Ideação do Tipo de Documento e Banco de Dados

### Contexto

No desenvolvimento de sistemas corporativos e industriais, uma das decisões mais importantes é **definir corretamente quais dados serão armazenados** e **qual tipo de banco de dados é mais adequado** para esse cenário. Essa escolha impacta diretamente fatores como desempenho, escalabilidade, segurança, custo e facilidade de manutenção da aplicação.

Nesta unidade curricular, vocês irão desenvolver, ao longo do semestre, um **projeto de banco de dados completo** dentro da metodologia **PBL (Problem Based Learning)**. A atividade desta aula marca o **início do projeto**, com foco na **compreensão do problema e ideação da solução**.


### Desafio da Atividade

Em grupos, vocês deverão **pesquisar e idealizar um cenário realista** de uma aplicação **industrial ou corporativa** e, a partir dele:

1. **Identificar qual tipo de documento, informação ou dado será armazenado na aplicação**
2. **Analisar as características desses dados**
3. **Definir, de forma justificada, qual tipo de banco de dados seria mais adequado** para esse cenário


### Orientações para o Grupo

Cada grupo deverá escolher **um único cenário de aplicação**, por exemplo (apenas como inspiração):

* Sistema de controle de estoque
* Sistema de gestão de clientes
* Plataforma de pedidos ou vendas
* Sistema de monitoramento industrial
* Aplicação de recursos humanos
* Sistema de prontuários, relatórios ou registros técnicos

A partir do cenário escolhido, o grupo deve discutir e levantar:

#### 1. Tipo de Documento / Dado

Descreva:

* Que tipo de informação será armazenada (ex.: cadastros, registros, históricos, logs, documentos, etc.)
* Estrutura dos dados:

  * Dados estruturados (tabelas bem definidas)
  * Dados semi-estruturados (JSON, documentos)
  * Dados não estruturados (textos longos, imagens, logs)
* Volume estimado de dados (baixo, médio, alto)
* Frequência de acesso (consultas frequentes, gravações constantes, ambos)

#### 2. Tipo de Banco de Dados

Com base na análise anterior, indique:

* Qual abordagem parece mais adequada:

  * Banco de dados relacional
  * Banco de dados não-relacional (NoSQL)
* Justifique a escolha considerando:

  * Organização dos dados
  * Relacionamentos
  * Escalabilidade
  * Facilidade de consulta
  * Cenário de uso da aplicação

> **Obs.:** Não é necessário definir um SGBD específico (como MySQL, PostgreSQL, MongoDB, etc.) nesta etapa. O foco é **entender o problema e o tipo de solução**, não a tecnologia final.


### Entregável da Aula

Ao final da atividade, cada grupo deverá apresentar **brevemente** (oralmente ou por registro escrito, conforme orientação do professor):

* Nome ou descrição do cenário escolhido
* Tipo de documento/dados que a aplicação irá armazenar
* Tipo de banco de dados mais indicado e a justificativa da escolha
