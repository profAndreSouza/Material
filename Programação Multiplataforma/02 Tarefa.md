# Atividade Stack MING

## Objetivo da atividade

O vídeo deve mostrar **todo o fluxo de dados**, desde a **geração do sinal no dispositivo** até a **visualização final no Grafana**.

## Requisitos obrigatórios

O projeto apresentado no vídeo deverá conter os seguintes elementos:

* **Dispositivo físico ou simulado**

  * Pode ser utilizado um **dispositivo físico** (Arduino, ESP32 ou similar), caso vocês possuam.
  * Alternativamente, pode ser usado o **simulador Wokwi**.

* **MQTT**

  * Pode ser utilizado:
  * AWS IoT Core
  * Um broker MQTT hospedado em uma **instância EC2 da AWS**
  * Ou **HiveMQ**

* **Node-RED**

  * Pode ser utilizado:
  * AWS IoT Core
  * Ou hospedado em uma **instância EC2 da AWS**

* **InfluxDB**

  * Utilizar o **InfluxDB Serverless gratuito do InfluxData**

* **Grafana**

  * Pode ser utilizado:
  * AWS IoT Core
  * Ou hospedado em uma **instância EC2 da AWS**

## O que deve aparecer no vídeo

O vídeo deverá demonstrar claramente:

1. A **geração de dados no dispositivo** (sensor real ou simulado).
2. O **envio dos dados via MQTT**.
3. O **processamento ou roteamento no Node-RED**.
4. O **armazenamento no InfluxDB**.
5. A **visualização dos dados no Grafana**.

Ou seja, vocês devem mostrar **todo o pipeline funcionando**, do dispositivo até o dashboard.
